## How HQ fits together

```
Your browser  →  HQ UI (Next.js)  →  Your Supabase project  →  Gateway hosts  →  Agents
```

The **UI** is where you manage everything: agents, tasks, CRM, documents, automations, connections, and settings.

**Supabase** is the shared database and auth system — owned by you, on your account.

**Gateways** are Docker hosts that run agents. One gateway is the default; you can add more on other machines.

**Agents** are long-lived workspaces: their own git branch, Chrome profile, messaging channel, and skills.

## Open source by design

HQ is not a hosted product with a self-hosted checkbox — the repo is the product. The docs include UI paths, manual paths, operational notes, security risks, and extension points so you can run it yourself or build your own version.
